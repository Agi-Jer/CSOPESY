How to Compile and RunThis project requires a C++ compiler supporting C++11 or higher (e.g., g++ or clang++) due to its heavy reliance on native multi-threading features (, , ).1. CompilationUsing a Standard GCC Compiler (Linux / macOS Terminal / Windows MinGW):Execute the following command in your terminal to compile all source files into a unified executable:Bashg++ -std=c++11 -pthread *.cpp -o csopesy_sim
Note: The -pthread flag is mandatory to enable multi-threading linkers on Unix-based systems.2. ExecutionOnce compiled successfully, invoke the binary executable from your terminal interface:On Linux / macOS:Bash./csopesy_sim
On Windows (Command Prompt / PowerShell):DOScsopesy_sim.exe
Command Console ReferenceUpon booting up, the system runs an interactive shell environment supporting the following structural commands:CommandAction / Technical BehaviorinitializeBoots up simulation systems, reads configuration parameters, and spawns the core worker threads.screen -s Instantiates a completely new process payload and drops the interface into its local terminal view.screen -r Reattaches the active viewport into an existing process's private log history and running display.screen -lsCompiles a live system data table capturing state, timestamps, and execution tracking counters across all four queues.scheduler-testWakes an asynchronous background thread to continuously flood the Ready Queue with new processes every x cycles.report-utilCompiles an overall system state metadata snapshot and flushes the formatted payload to an external csopesy_log.txt file.exitDestroys local process terminal contexts to step back into the primary command interpreter shell.
